## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** "Rate of change" — how fast glucose is rising or falling right now — is a key forecast input. bgdude computes it crudely, and computes it slightly differently when training the model versus when running live; it also doesn't account for the noisy first day of a new sensor.

**Reason for change.** A robust rate-of-change used identically in training and live, plus down-weighting a sensor's noisy first 24 hours, improves every forecast horizon. Worth doing first in the forecasting work.

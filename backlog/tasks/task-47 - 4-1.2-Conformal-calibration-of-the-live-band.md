## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** Even a well-shaped uncertainty band is only useful if its width matches reality — if the band says "90% confident", the real value should land inside it about 90% of the time. bgdude's band can currently only be widened, never tightened, when it turns out mis-sized.

**Reason for change.** "Conformal calibration" is a standard technique that rescales the band from its recent track record so its stated confidence matches reality in both directions, while keeping the lopsided shape from the previous task.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
**Technical notes.** Extend UncertaintyCalibrator: per-horizon ratio scores |actual−predicted|/halfWidth over 14 d of reconciled predictions → band scale = ⌈(n+1)(1−α)⌉/n empirical quantile (α=0.10), scoring each side separately (preserves 4-1.1 asymmetry). Keep a floor (≥0.7×) so a lucky fortnight can't cause overconfidence. minSamples=20 guard stays; recompute in updateRecentForecastError.

**Testing.** Synthetic triples hit 86–94% coverage from both directions; the floor engages when data is favourable. ML-honesty tests (coverage + bias, synthetic recovery); guard the forecaster promotion gate; `flutter analyze`/`flutter test` green.

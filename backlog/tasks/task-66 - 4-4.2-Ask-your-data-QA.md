## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** There's no way to ask plain-English questions about your own data, like "how do burritos usually treat me?"

**Reason for change.** A question-and-answer feature that retrieves facts the app has already computed and has the AI only phrase them — citing the exact fact behind each statement — adds real daily value while refusing to invent numbers.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Retrieval over computed facts by keyword/date
- [ ] #2 LLM phrases + cites backing fact (reject uncited)
- [ ] #3 Fixed question taxonomy
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
- Define a fixed question taxonomy first.
- Build retrieval over COMPUTED facts (narratives, reports, metrics, events) selected by keyword/date parsing.
- The LLM only phrases the retrieved facts and must cite which fact backs each claim; reject uncited claims.
- Tests: retrieval-selection tests; reject-uncited-claim test; taxonomy coverage; validation/grounding tests (bounds + OCR-grounding); degrade gracefully with no model.
- Verify: `flutter analyze` clean, `flutter test` green.
<!-- SECTION:PLAN:END -->

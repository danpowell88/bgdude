## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** "Nightscout" is a popular open-source personal server many people with diabetes already use to collect their CGM and pump data. bgdude can't currently read from it.

**Reason for change.** A "follower mode" that pulls from Nightscout lets bgdude run off that data (or blend it in) without importing the same reading twice — a common request and the groundwork for other data sources.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
**Technical notes.** Add fetchEntries/fetchTreatments(since) to NightscoutClient; a pull loop in jobs behind a source-mode setting; merge through the backfill dedupe path keyed on NS _id; source priority (pump > NS) with per-sample provenance so dual-live never double-ingests.

**Testing.** Repo test: NS entries merge without duplicating pump data; provenance + source-priority respected. Mock NS responses. Repository tests on `NativeDatabase.memory()`; drift schema-export + migration tests before any schema change.

## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** Before promoting a newly-trained forecast model into daily use, bgdude checks it on held-out data using one simple split, and it keeps no history to roll back a bad model. Because nearby glucose readings are so similar, that single split can be fooled.

**Reason for change.** A rigorous "walk-forward" check (several rolling test windows with a gap to prevent leakage) gives an honest verdict, and keeping the last few models lets a bad promotion be undone. This task also hosts the GBM early-stopping (P2-4) and the validation upgrade (P2-2).
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
- Prerequisite: TASK-2 (P0-2).
- Implement K=3–4 contiguous blocked folds with a purge gap of maxHorizon (±6 min label slop makes the gap mandatory).
- Pooled fold metrics gate promotion; final model retrains on everything; <~10 days falls back to the single split.
- Persist `foldRmses` in `ModelRuns` metrics JSON.
- Validation-chosen `nEstimators` (early stopping — P2-4).
- Rollback: store the last ~5 promoted blobs; Advanced-screen "Model history" list with restore (refuses on feature-version mismatch).
- Tests: purge-gap leakage assertion; pooled-gate test; restore round-trip (refuses on version mismatch); ML-honesty tests (coverage + bias, synthetic recovery); guard the forecaster promotion gate.
- Verify: `flutter analyze` clean, `flutter test` green.

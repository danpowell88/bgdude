## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** bgdude personalises its forecast by learning a correction from your own history — but a brand-new user has no history, so for the first weeks the forecast falls back to generic physics only.

**Reason for change.** Shipping a "population prior" (a model pre-trained on many simulated users) warm-starts accuracy from day one, then gradually blends toward your personal model as your own data accumulates — better cold-start without overfitting.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
- Prerequisite: TASK-2 (P0-2).
- Train per-horizon priors offline on ~100 simulator variants (`dev/sim_data.dart`, varied dawn/overshoot via a `tools/` script) against the same baseline + feature layout.
- Ship `assets/models/residual_prior_v<ver>.json` (assert version inside; refuse on mismatch; regenerate on every feature bump).
- Add `BlendedResidualModel`: residual=(1−w)·prior+w·personal, sigma=max(prior,personal), w=(personalSamples/600).clamp(0,1).
- Make the promotion gate incumbent the blend.
- Advanced screen shows prior/blended(w)/personal.
- Tests: blend-math unit test; asset version-mismatch refusal; cold-start integration check; ML-honesty tests (coverage + bias, synthetic recovery); guard the forecaster promotion gate.
- Verify: `flutter analyze` clean, `flutter test` green.

## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** The pump reports a malfunction status — a set of flags that are all zero when healthy. bgdude can read it. (Note: during the exploration the actual message turned out to be "MalfunctionBitmaskStatus", not the name originally assumed.) These read messages were confirmed to return real data during the July 2026 live-pump exploration (the Protocol Explorer sweep); their exact byte layouts are recorded in doc/pump-protocol.md.

**Reason for change.** Polling this and raising a high-priority alert on any non-zero flag turns the app into a proactive safety net for pump malfunctions.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
**Technical notes.** MalfunctionBitmaskStatus op 119 (bitmask; 0 = none). NOTE the class is MalfunctionBitmaskStatusResponse, not MalfunctionStatus. Poll on connect + each refresh; any non-zero bit → a high-priority notification (route through the §3.C alert path).

**Testing.** Decode/bitmask test; a non-zero bit fires the notification. Add a decode branch + snapshot fields; unit-test the decoder against the captured cargo in doc/pump-protocol.md; on-device, reconcile the value against what the pump screen shows. Update doc/user-guide.html.

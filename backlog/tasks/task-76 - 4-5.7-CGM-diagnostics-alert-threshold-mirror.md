## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**Background.** The pump exposes CGM diagnostics — the transmitter's hardware id (which decodes to a short readable code), extended sensor status, and the pump's own high/low and rate alert thresholds. These read messages were confirmed to return real data during the July 2026 live-pump exploration (the Protocol Explorer sweep); their exact byte layouts are recorded in doc/pump-protocol.md.

**Reason for change.** A CGM diagnostics card plus mirroring the pump's alert thresholds lets bgdude's alerts line up with the pump's, and gives useful troubleshooting info.
